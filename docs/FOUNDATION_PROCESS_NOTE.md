# Foundation Process Note

Diese Notiz fixiert die Reihenfolge der Foundation-Phase:

1. UI/UX-Harmonisierung zuerst (Design-System + gemeinsame Komponenten).
2. CI-Stabilität und Secrets-Härtung bleiben kritisch, laufen aber getrennt von UI-Refactors.
3. Pilot/Beta erst nach stabiler Foundation und QA.
4. Compliance/Marktforschung werden als eigene Artefakte geführt und nicht mit UI-PRs vermischt.
