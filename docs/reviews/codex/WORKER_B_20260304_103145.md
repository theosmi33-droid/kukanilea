## 0) Branch-Sync

### Ergebnis
- Lokales Repository hat **nur einen Branch `work`**.
- Es gibt **keinen lokalen `main` Branch** und **kein `origin` Remote**.
- Domain-Branches (`codex/messenger`, `codex/emailpostfach`, `codex/floating-widget-chatbot`) sind in dieser Umgebung nicht vorhanden.

### Auswirkung
- Ein klassischer Branch-Sync (rebase/merge mit `main` bzw. remote tracking) ist in dieser Umgebung **nicht vollständig verifizierbar**.
- Risiko: Divergenz kann außerhalb dieser Session unentdeckt bleiben.

### Empfehlung (P1)
1. Remote hinzufügen (`origin`) und `main` abrufbar machen.
2. Domain-Branches lokal auschecken/aktualisieren.
3. Branch-Sync-Check standardisieren (`git fetch --all --prune && git branch -vv`).

## 3) Domain: floating-widget-chatbot

### 3.1 Overlap + Domain-Checks
- `check_domain_overlap.py --reiter floating-widget-chatbot ...` => **DOMAIN_OVERLAP_DETECTED**.

**Befund-Details:**
- Allowlist im Script erwartet: `app/templates/partials/floating_chat.html`, `app/static/js/chatbot.js`.
- Scope-Dokument nennt andere Pfade (`layout.html`, `partials/chat_widget.html`, `chat_widget.js`, `chat_widget.css`).
- In Repo ist nur `app/static/js/chatbot.js` vorhanden; die anderen Scope-Dateien fehlen.

**Bewertung:** Governance-Mismatch zwischen Scope-Doku und Overlap-Checker/Realität.

### 3.2 Confirm-Gate Prüfung
- In `app/static/js/chatbot.js` existiert ein Confirm-Flow (`requires_confirm`, `pending_id`, `confirm=true`).
- In `layout.html` gibt es zusätzlich einen Legacy-Prompt-Flow (`prompt('CONFIRM')`) für `data.actions`.

**Bewertung:** Confirm-Gate grundsätzlich vorhanden; allerdings **zwei parallele UX-Pfade** (Compact-Chat vs. Layout-Chat).

### 3.3 Offline-Fähigkeit
- Request-Payload an `/api/chat/compact` enthält `offline: !navigator.onLine`.
- Layout setzt Status visuell auf `KI (OFFLINE)` bei Disconnect.

**Bewertung:** Offline-Zustand wird propagiert und visualisiert.

### 3.4 Risiken
- **P1:** Scope-/Allowlist-Drift erzeugt false positives oder blockiert legitime Arbeit.
- **P1:** Doppelter Confirm-Mechanismus kann inkonsistentes Verhalten erzeugen.
- **P2:** Legacy-`prompt('CONFIRM')` in Layout ist schwächer als zentral orchestrierter Pending-ID-Flow.
