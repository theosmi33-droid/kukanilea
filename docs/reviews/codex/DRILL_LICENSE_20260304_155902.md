# Restore Drill & License Validation Report

## Scope
- Restore drill with deterministic tenant seed data.
- Backup/restore degraded mode when NAS/SMB unavailable.
- License state machine transitions: `active`, `grace`, `blocked`, `recover`.
- Read-only enforcement for write operations during blocked/recover states.

## Operator Results
- RTO/RPO are now emitted by backup/restore scripts in operator report files (`rto_seconds`, `rpo_seconds`).
- Restore validation compares pre/post snapshots using per-table counts and sample row hashes.

## Critical Transition Audits
- `license_transition` event emitted when runtime state resolves from signed license payload.
- `license_write_blocked` event emitted on denied write requests in read-only mode.

## Drill Procedure
1. `python3 scripts/seed_demo_data.py`
2. `python3 scripts/ops/restore_validation.py --phase before`
3. `./scripts/ops/backup_to_nas.sh`
4. `./scripts/ops/restore_from_nas.sh`
5. `python3 scripts/ops/restore_validation.py --phase after`

## Conclusion
- Deterministic seeded tenant data enables reproducible proof of restore integrity.
- NAS outages no longer hard-fail operations; system enters explicit degraded mode.
- License transitions are explicit/tested and write-path lockout is enforced in app request lifecycle.
