## ORCH-20260305-001 — Observer auth DB fallback hardening

- **Lane:** dashboard
- **Scope:** `app/core/observer.py`, `tests/test_observer_outbound_queue.py`
- **Root cause:** observer outbound queue stats used a developer-specific absolute fallback path (`/Users/.../auth.sqlite3`), causing non-portable metrics collection in CI/server environments.
- **Change summary:** added `_resolve_auth_db_path()` to prioritize `KUKANILEA_AUTH_DB`, then repo-local `instance/auth.sqlite3`, then legacy path for backward compatibility.
- **Reliability impact:** outbound queue metric reads now auto-discover local instance DB in shared environments, reducing silent zero-metric degradation.
